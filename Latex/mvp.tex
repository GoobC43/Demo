\section*{1. MVP Objective}

Build a fully functional maritime disruption decision-support system for U.S.-based mid-market electronics manufacturers.

The MVP must:

\begin{itemize}
    \item Detect a maritime disruption event
    \item Map exposure to inbound SKUs
    \item Compute inventory depletion risk
    \item Simulate 3 mitigation strategies
    \item Output financially optimized recommendation
    \item Generate actionable mitigation drafts
\end{itemize}

No unnecessary features. No overengineering.

\section*{2. What We ARE Building}

\subsection*{Core Features}

\begin{enumerate}
    \item Manual + API-triggered disruption input
    \item Inventory runway calculation
    \item Revenue-at-risk computation
    \item Strategy comparison engine (3 strategies)
    \item Constrained optimization logic
    \item LLM-generated mitigation drafts
    \item Simple web dashboard
\end{enumerate}

\subsection*{What We Are NOT Building}

\begin{itemize}
    \item Full ERP integration
    \item Multi-agent orchestration
    \item Reinforcement learning
    \item Real-time streaming architecture
    \item Advanced ML prediction
\end{itemize}

\section*{3. User Workflow}

\subsection*{Step 1: Disruption Trigger}

User inputs or system detects:

\begin{verbatim}
Port: Los Angeles
Disruption Type: Labor Strike
Expected Delay: 12 days
Severity: 0.8
\end{verbatim}

\subsection*{Step 2: Exposure Mapping}

System computes:

\begin{itemize}
    \item Shipments routed via LA
    \item SKUs inside affected containers
    \item Inventory runway per SKU
    \item Stockout prediction
\end{itemize}

\subsection*{Step 3: Revenue-at-Risk}

For SKU $i$:

\[
Runway_i = \frac{Inventory_i}{DailyDemand_i}
\]

If:

\[
Runway_i < ExpectedDelay
\]

Then:

\[
RevenueAtRisk_i = Demand_i \times Price_i \times DelayGap
\]

Total:

\[
TotalRevenueAtRisk = \sum_i RevenueAtRisk_i
\]

\subsection*{Step 4: Strategy Simulation}

Three strategies:

\begin{itemize}
    \item $S_1$: Do Nothing
    \item $S_2$: Air Freight All
    \item $S_3$: Split (60\% Reroute, 20\% Air, 20\% Buffer)
\end{itemize}

For each strategy compute:

\[
NetImpact_s = RevenuePreserved_s - MitigationCost_s
\]

System selects:

\[
S^* = \arg\max_s NetImpact_s
\]

\section*{6. Optimization Logic (Simplified)}

\subsection*{Revenue Preserved}

\[
RevenuePreserved_s = TotalRevenueAtRisk - ResidualLoss_s
\]

\subsection*{Mitigation Cost}

\[
MitigationCost_s =
AirCost_s + RerouteCost_s + HoldingCost_s
\]

\subsection*{Constraint: Working Capital}

\[
MitigationCost_s \leq WorkingCapitalLimit
\]

Dashboard must display:

\begin{itemize}
    \item Disruption summary

    \item Recommended action
    \item Generated drafts

\section*{9. MVP Success Criteria}

The system is considered functional if:

\begin{itemize}
    \item Disruption input produces exposure report
    \item Revenue-at-risk is computed correctly
    \item Three strategies are compared numerically
    \item Optimization selects highest net-impact strategy
    \item LLM generates actionable drafts
    \item All results display cleanly on dashboard
\end{itemize}

\section*{10. MVP Definition of Value}

The MVP must allow a VP of Supply Chain to answer:

\begin{quote}
"If Port of Los Angeles shuts down for 12 days,
what should I do right now,
and what is the financial impact?"
\end{quote}

If the system answers this clearly, it is useful.
